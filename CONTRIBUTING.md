# 参与贡献

欢迎为 TJHub 做出贡献！本文档说明如何参与项目。

## 基本原则

- TJHub 是一个**非官方**校园信息导航项目
- 内容贡献遵循**社区维护**模式
- 所有提交需经过**人工审核**
- 遵循中文为主要交流语言

## 提交 Issue

以下场景请提交 Issue（使用对应的 Issue 模板）：

### 新增入口

如果希望推荐一个适合 TJHub 收录的网站或工具入口，请使用"新增网站或工具入口"模板。需提供：

- 名称与链接
- 用途说明
- 来源性质（官方网站 / 学生项目 / 第三方工具）

### 反馈失效链接

如果发现某个入口链接失效、跳转错误或信息过期，请使用"失效链接反馈"模板。需提供：

- 出问题的条目名称
- 失效链接
- 问题说明（无法访问、跳转错误、内容过期等）

### 学生项目或友链申请

如果维护了一个适合同学校园场景使用的项目（工具站、资料站等），请使用"学生项目或友链申请"模板。需提供：

- 项目名称与链接
- 维护者信息
- 项目说明
- 隐私、登录或版权风险说明

### 指南内容建议

如果希望新增或修改站内指南，请使用"指南内容建议"模板。需提供：

- 指南主题
- 适用对象
- 建议内容

## 提交 Pull Request

### 内容编辑

如果你熟悉 Markdown 和 Git，可以直接修改 `content/` 目录下的文件并提交 PR。

内容文件结构：

```
content/
├── pages/            # 静态页面（Markdown）
├── guides/           # 站内指南（Markdown）
├── collections/
│   ├── links/        # 入口链接（YAML frontmatter 格式）
│   └── projects/     # 学生项目（YAML frontmatter 格式）
```

### 代码贡献

如果你希望改进网站功能：

1. 先提出 Issue 讨论你的想法
2. Fork 本仓库并创建功能分支
3. 实现变更，确保 `npm run build` 和 `npm run test` 通过
4. 提交 PR，描述改动内容和原因

### 内容编辑规范

- 所有内容文件使用 Markdown 格式，入口和项目条目在 frontmatter 中定义
- `slug` 字段必须唯一
- `title` 字段使用中文
- `url` 字段必须是可访问的有效链接
- 为条目添加 `aliases` 以便搜索
- 分类标签应使用已有类别，添加新类别需在 PR 中说明理由

## 审核流程

### 审核状态

条目使用 `reviewStatus` 字段控制审核状态：

| 状态 | 说明 |
|------|------|
| `draft` | 草稿，未提交审核 |
| `pending` | 等待审核 |
| `approved` | 已通过审核，公开展示 |
| `rejected` | 已驳回 |

### 审核标准

审核时关注以下方面：

1. **准确性**：链接是否有效，信息是否准确
2. **适用性**：内容是否与同学校园场景相关
3. **安全性**：目标网站是否存在已知安全风险
4. **合规性**：是否涉及侵权、隐私侵犯或不适当内容
5. **规范性**：格式是否符合内容规范

### 审核周期

- 一般 Issue 提交会在 **1-3 个工作日**内得到回复
- PR 审核由项目维护者适时处理

## 仓库管理规范

- `main` 分支为生产部署分支，直接推送受限
- 所有变更通过 PR 合入 `main`
- 每个 PR 需通过 CI 检查（构建和测试）
- 提交信息清晰描述变更内容
